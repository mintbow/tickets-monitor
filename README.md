# Монитор дешёвых авиабилетов из Екатеринбурга

Скрипт постоянно следит за ценами на авиабилеты из **Екатеринбурга (SVX)** на
пляжные направления, куда гражданам РФ не нужна виза (достаточно
загранпаспорта), и присылает уведомление в **Telegram**, когда находит билет
**заметно дешевле обычной цены**.

- Приоритет — билеты **туда-обратно** с интервалом около двух недель.
- Если выгодного «туда-обратно» нет, но есть дешёвый билет **в одну сторону** —
  уведомление тоже придёт.
- Данные берутся из официального **Travelpayouts / Aviasales Data API** (не
  парсинг сайта — это надёжнее и не ломается от анти-бот защиты).

## Направления

Сочи, Анталья, Стамбул, Хургада, Шарм-эль-Шейх, Пхукет, Бангкок, Нячанг,
Хошимин, Дубай, Коломбо. Список легко менять в `routes.py`.

## Что нужно подготовить

### 1. Токен Travelpayouts (бесплатно)

1. Зарегистрируйтесь на https://www.travelpayouts.com/
2. В личном кабинете найдите раздел с **API-токеном** и скопируйте его.

### 2. Telegram-бот

1. Откройте в Telegram чат с **@BotFather**.
2. Отправьте `/newbot`, придумайте имя и username бота.
3. BotFather пришлёт **токен бота** вида `123456789:AA...` — это `TELEGRAM_BOT_TOKEN`.
4. Напишите своему новому боту любое сообщение (например, «привет») — это нужно,
   чтобы бот мог вам отвечать.
5. Узнайте свой **chat_id**: откройте в браузере
   `https://api.telegram.org/bot<ВАШ_ТОКЕН>/getUpdates`
   (подставьте токен). В ответе найдите `"chat":{"id":...}` — это число и есть
   `TELEGRAM_CHAT_ID`.
   Либо напишите боту **@userinfobot** — он пришлёт ваш id.

## Установка

```powershell
# 1. Установить зависимости
pip install -r requirements.txt

# 2. Создать файл .env из шаблона и заполнить токены
copy .env.example .env
notepad .env
```

В `.env` впишите три значения:

```
TRAVELPAYOUTS_TOKEN=ваш_токен
TELEGRAM_BOT_TOKEN=токен_бота
TELEGRAM_CHAT_ID=ваш_chat_id
```

## Запуск

```powershell
# Проверить, что Telegram работает — придёт тестовое сообщение
python main.py --test-notify

# Один прогон: посмотреть в консоли самые дешёвые билеты и записать историю
python main.py --once

# Постоянный мониторинг (проверка раз в полчаса), остановка — Ctrl+C
python main.py
```

## Автозапуск (уже настроен)

В Планировщике задач Windows создана задача **AviasalesMonitor**, которая
**каждые 30 минут** скрыто (без окна консоли) запускает один прогон. Цепочка:
`run_monitor.vbs` (прячет окно) → `run_monitor.bat` (запускает Python) →
`main.py --once`. Вывод пишется в `monitor.log`.

Управление задачей (PowerShell):

```powershell
# посмотреть состояние и время следующего запуска
Get-ScheduledTaskInfo -TaskName "AviasalesMonitor"

# запустить прямо сейчас (не дожидаясь расписания)
Start-ScheduledTask -TaskName "AviasalesMonitor"

# временно отключить / включить
Disable-ScheduledTask -TaskName "AviasalesMonitor"
Enable-ScheduledTask  -TaskName "AviasalesMonitor"

# удалить автозапуск полностью
Unregister-ScheduledTask -TaskName "AviasalesMonitor" -Confirm:$false
```

Задача работает, пока вы вошли в систему (данные Aviasales всё равно кэшируются,
поэтому проверки раз в полчаса достаточно). Чтобы монитор работал 24/7 даже при
выключенном компьютере — см. раздел «Развёртывание в облаке» ниже.

> Свежий лог: последние строки — `Get-Content monitor.log -Tail 20 -Encoding UTF8`.

## Настройка

Все пороги и интервалы — в `config.py`:

| Параметр | Значение по умолчанию | Что делает |
|---|---|---|
| `DROP_THRESHOLD` | `0.25` | насколько ниже медианы = «дёшево» (25%) |
| `TRIP_DURATION_DAYS` | `14` | желаемая длительность поездки |
| `MONTHS_AHEAD` | `6` | на сколько месяцев вперёд искать |
| `CHECK_INTERVAL_SECONDS` | `1800` | как часто проверять при ручном `python main.py` (сек.) |
| `ALERT_COOLDOWN_HOURS` | `12` | не повторять одно уведомление чаще |
| `MIN_OBSERVATIONS` | `5` | минимум истории для расчёта «обычной» цены |

Абсолютные пороги «дёшево в любом случае» (`hard_cap_rt`) для каждого
направления — в `routes.py`. Их стоит подстроить под свои ожидания по ценам.

## Как определяется «дёшево»

1. Скрипт копит историю цен в `data/prices.db` (SQLite).
2. «Обычная» цена маршрута = медиана наблюдений за последние 3 недели.
3. Уведомление отправляется, если найденная цена **на 25%+ ниже медианы**
   **или** ниже абсолютного порога направления (`hard_cap`). Абсолютный порог
   нужен, чтобы уведомления шли уже в первые дни, пока истории мало.

> Примечание: API отдаёт кэшированные цены реальных поисков. Итоговую цену и
> наличие мест всегда проверяйте по ссылке на Aviasales перед покупкой.

## Развёртывание в облаке (работа 24/7 без включённого ПК)

Текущий автозапуск работает, пока включён компьютер. Чтобы монитор крутился
круглосуточно независимо от ПК:

| Вариант | Цена | Комментарий |
|---|---|---|
| **GitHub Actions** (cron) | бесплатно | Запуск по расписанию на серверах GitHub; историю цен храним в репозитории. Рекомендуется для этой задачи. |
| **Oracle Cloud Free Tier** | бесплатно | Небольшой Linux-сервер 24/7; ставим скрипт как есть. |
| **VPS** (Timeweb, Aeza и т.п.) | ~150–400 ₽/мес | Простой арендованный сервер. |
| **Raspberry Pi / мини-ПК** | разовая покупка | Стоит дома, мало энергии. |

Ключи (`TRAVELPAYOUTS_TOKEN`, `TELEGRAM_BOT_TOKEN`, `TELEGRAM_CHAT_ID`) в облаке
задаются через «секреты», а не в коде.

### GitHub Actions — пошагово (рекомендуемый бесплатный путь)

В проекте уже готовы:
- `.github/workflows/monitor.yml` — расписание (каждые 30 минут) и запуск в облаке;
- `deploy_to_github.ps1` — скрипт, который создаёт репозиторий и переносит ключи.

Как безопасно устроены ключи: файл `.env` **не** попадает в репозиторий (он в
`.gitignore`). Скрипт читает ключи из `.env` на вашей машине и кладёт их в
**зашифрованные секреты GitHub** — в коде их нет.

Про приватность репозитория:
- **Public (рекомендуется):** минуты GitHub Actions бесплатны без ограничений.
  Код виден всем, но секретов в коде нет — это безопасно.
- **Private:** код скрыт, но бесплатно только 2000 минут/мес — при проверке раз
  в 30 минут этого не хватит. Тогда лучше увеличить интервал (`cron` в
  `monitor.yml`) до 1–2 часов.

Шаги (один раз):

```powershell
# 1. Установить GitHub CLI (если ещё нет) и перезапустить терминал
winget install --id GitHub.cli

# 2. Войти в свой GitHub (откроется браузер; разрешите доступ к 'workflow')
gh auth login

# 3. Из папки проекта — запустить деплой (public по умолчанию)
powershell -ExecutionPolicy Bypass -File .\deploy_to_github.ps1
#   для приватного репозитория:
#   powershell -ExecutionPolicy Bypass -File .\deploy_to_github.ps1 -Visibility private
```

После этого монитор работает в облаке 24/7 без вашего ПК. Первый прогон можно
запустить сразу, не дожидаясь расписания: вкладка **Actions** в репозитории →
workflow **Aviasales price monitor** → **Run workflow** (или
`gh workflow run "Aviasales price monitor"`).

История цен хранится в самом репозитории (файл `data/prices.db`, обновляется
после каждого прогона), поэтому расчёт «обычной» цены не теряется между запусками.

> Если запускаете монитор в облаке, локальную задачу автозапуска можно отключить,
> чтобы уведомления не дублировались:
> `Disable-ScheduledTask -TaskName "AviasalesMonitor"`.
