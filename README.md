# Монитор дешёвых авиабилетов из Екатеринбурга

Следит за ценами на авиабилеты из **Екатеринбурга (SVX)** на пляжные направления,
куда гражданам РФ не нужна виза (достаточно загранпаспорта), и присылает
уведомление в **Telegram**, когда находит билет **заметно дешевле обычной цены**.

- Приоритет — билеты **туда-обратно** с интервалом около двух недель; если
  выгодного «туда-обратно» нет, но есть дешёвый билет **в одну сторону** —
  уведомление тоже придёт.
- Данные берутся из официального **Travelpayouts / Aviasales Data API**.
- Работает **в облаке** (GitHub Actions) 24/7 без включённого компьютера, а при
  необходимости — **локально** на вашем ПК.

## Как определяется «дёшево»

1. При каждом прогоне цены пишутся в базу истории `data/prices.db` (SQLite).
2. «Обычная» цена маршрута = медиана наблюдений за последние ~3 недели.
3. Уведомление отправляется, если найденная цена **на 25%+ ниже медианы**
   **или** ниже абсолютного порога направления (`hard_cap` в `routes.py`).
   Абсолютный порог нужен, чтобы уведомления шли уже в первые дни, пока истории
   мало.

> Итоговую цену и наличие мест всегда проверяйте по ссылке на Aviasales перед
> покупкой — API отдаёт кэшированные цены реальных поисков.

## Работа с облаком (GitHub Actions)

Монитор развёрнут в репозитории **https://github.com/mintbow/tickets-monitor** и
запускается сам **каждые 30 минут** на серверах GitHub. Ничего держать включённым
не нужно.

### Где смотреть прогоны и логи

- Открой репозиторий → вкладка **Actions** → workflow **Aviasales price monitor**.
- Каждый запуск — отдельная строка. Зелёная галочка = успех, красный крест =
  ошибка. Кликни по запуску → job **check-prices** → раскрой шаг **Run monitor**,
  чтобы увидеть найденные цены и какие уведомления отправлялись.
- То же из терминала (нужен установленный GitHub CLI `gh`):
  ```powershell
  gh run list   -R mintbow/tickets-monitor
  gh run view --log -R mintbow/tickets-monitor   # логи последнего запуска
  ```

### Запустить прогон вручную (не дожидаясь расписания)

- На сайте: **Actions** → **Aviasales price monitor** → кнопка **Run workflow**.
- Или из терминала:
  ```powershell
  gh workflow run monitor.yml -R mintbow/tickets-monitor
  ```

### Как что-то изменить и сохранить

Изменения вступают в силу со следующего прогона. Есть два способа.

**Способ A. Правка на своём ПК и загрузка (git push).** Подходит для любых правок.

```powershell
cd "C:\Users\ssaas\OneDrive\Документы\Claude\проект 1 - парсер авиабилетов"
# ... отредактируйте нужные файлы ...
git add -A
git commit -m "Короткое описание изменения"
git push
```

**Способ B. Правка прямо на сайте GitHub.** Удобно для мелких изменений.

1. Открой файл в репозитории (например `config.py` или `routes.py`).
2. Нажми значок карандаша (**Edit this file**).
3. Внеси правку → внизу **Commit changes**.

### Частые изменения

| Что поменять | Где | Как |
|---|---|---|
| **Как часто проверять** | `.github/workflows/monitor.yml` | строка `cron: "*/30 * * * *"`. Например `"0 * * * *"` — раз в час. Время в **UTC** (на интервал не влияет). |
| **На сколько месяцев вперёд искать** | `config.py` | `MONTHS_AHEAD` (сейчас `2`). |
| **Направления** | `routes.py` | добавь/убери строки `Destination(...)`. |
| **Пороги «дёшево»** | `routes.py` | числа `hard_cap_rt` / `hard_cap_ow` у направления. |
| **Порог скидки, окно истории и т.п.** | `config.py` | см. раздел [Настройки](#настройки). |

После правки — сохрани способом A или B (см. выше).

### Как изменить ключи (секреты) в облаке

Ключи хранятся в **зашифрованных секретах** репозитория, а не в коде. Поменять:

- На сайте: репозиторий → **Settings** → **Secrets and variables** → **Actions** →
  нужный секрет (`TRAVELPAYOUTS_TOKEN`, `TELEGRAM_BOT_TOKEN`, `TELEGRAM_CHAT_ID`) →
  **Update**.
- Или из терминала:
  ```powershell
  gh secret set TELEGRAM_CHAT_ID -R mintbow/tickets-monitor --body "12345678"
  ```

## Добавить ещё одного получателя в Telegram

Бот может слать уведомления **сразу нескольким людям**. Для каждого получателя
нужен его личный `chat_id`; все id перечисляются в одном секрете `TELEGRAM_CHAT_ID`
**через запятую**.

Шаги, чтобы подключить нового человека:

1. **Новый человек пишет боту.** Он находит бота в Telegram по его username и
   отправляет любое сообщение (например «привет»). Это обязательно — Telegram не
   разрешает боту писать первым тому, кто с ним не начинал диалог.
2. **Узнаём его `chat_id`.** Самый простой путь: новый человек пишет боту
   **@userinfobot** — тот присылает его цифровой id.
   (Альтернатива: владелец бота открывает в браузере
   `https://api.telegram.org/bot<ТОКЕН_БОТА>/getUpdates` и находит там
   `"chat":{"id": ...}` нового человека.)
3. **Добавляем id к уже существующим через запятую.** Например, было `12345678`,
   стало `12345678,87654321`:
   - **Облако:** обнови секрет `TELEGRAM_CHAT_ID` (см. предыдущий раздел), вписав
     все id через запятую.
   - **Локально:** в файле `.env` строка
     `TELEGRAM_CHAT_ID=12345678,87654321`.
4. Готово — со следующего прогона уведомления пойдут всем перечисленным.

> Хотите слать в **групповой чат**? Добавьте бота в группу и используйте `chat_id`
> группы (он отрицательный, например `-1001234567890`). Узнать его можно тем же
> `getUpdates` после отправки сообщения в группе.

Проверить, что всё подключилось, можно тестовым сообщением (локально):
`python main.py --test-notify` — оно придёт всем получателям.

## Локальный запуск (резерв на случай отключения облака)

Если облако когда-нибудь отключат или захочется гонять монитор со своего ПК —
всё работает и локально.

### 1. Установка

```powershell
cd "C:\Users\ssaas\OneDrive\Документы\Claude\проект 1 - парсер авиабилетов"
pip install -r requirements.txt
```

### 2. Ключи в файле `.env`

Скопируй шаблон и впиши значения (как их получить — см.
[Как получить ключи](#как-получить-ключи-токены)):

```powershell
copy .env.example .env
notepad .env
```

```
TRAVELPAYOUTS_TOKEN=ваш_токен
TELEGRAM_BOT_TOKEN=токен_бота
TELEGRAM_CHAT_ID=ваш_chat_id        # можно несколько через запятую
```

### 3. Запуск

```powershell
# Проверить Telegram — придёт тестовое сообщение всем получателям
python main.py --test-notify

# Один прогон: показать в консоли самые дешёвые билеты и записать историю
python main.py --once

# Постоянный мониторинг (проверка раз в полчаса), остановка — Ctrl+C
python main.py
```

### 4. Автозапуск на Windows (Планировщик задач)

Автозапуск уже был настроен ранее задачей **AviasalesMonitor** (скрытый прогон
каждые 30 минут, лог в `monitor.log`). **Сейчас она отключена**, чтобы не
дублировать облачные уведомления. Управление:

```powershell
# включить обратно (если решите работать локально вместо облака)
Enable-ScheduledTask  -TaskName "AviasalesMonitor"

# снова отключить
Disable-ScheduledTask -TaskName "AviasalesMonitor"

# состояние и время следующего запуска
Get-ScheduledTaskInfo -TaskName "AviasalesMonitor"

# запустить прямо сейчас
Start-ScheduledTask   -TaskName "AviasalesMonitor"

# удалить автозапуск полностью
Unregister-ScheduledTask -TaskName "AviasalesMonitor" -Confirm:$false
```

> Не запускайте одновременно облако и локальную задачу — иначе уведомления будут
> приходить дважды (у них ещё и разные базы истории цен).

Свежий локальный лог: `Get-Content monitor.log -Tail 20 -Encoding UTF8`.

## Настройки

Пороги и интервалы — в `config.py`:

| Параметр | Значение | Что делает |
|---|---|---|
| `MONTHS_AHEAD` | `2` | на сколько месяцев вперёд искать |
| `DROP_THRESHOLD` | `0.25` | насколько ниже медианы = «дёшево» (25%) |
| `TRIP_DURATION_DAYS` | `14` | желаемая длительность поездки |
| `TRIP_DURATION_TOLERANCE` | `3` | допуск ± дней вокруг длительности |
| `BASELINE_WINDOW_DAYS` | `21` | окно истории для «обычной» цены |
| `MIN_OBSERVATIONS` | `5` | минимум истории для расчёта медианы |
| `ALERT_COOLDOWN_HOURS` | `12` | не повторять одно уведомление чаще |
| `CHECK_INTERVAL_SECONDS` | `1800` | интервал при ручном `python main.py` (сек.) |

Абсолютные пороги «дёшево» (`hard_cap_rt` / `hard_cap_ow`) по каждому
направлению — в `routes.py`.

> В облаке интервал задаётся не `CHECK_INTERVAL_SECONDS`, а `cron` в
> `monitor.yml` (см. [Частые изменения](#частые-изменения)).

## Как получить ключи (токены)

### Travelpayouts (бесплатно)

1. Зарегистрируйся на https://www.travelpayouts.com/
2. В личном кабинете найди **API-токен** и скопируй — это `TRAVELPAYOUTS_TOKEN`.

### Telegram-бот

1. Напиши в Telegram боту **@BotFather** → команда `/newbot` → задай имя и
   username. BotFather пришлёт **токен бота** (`123456789:AA...`) — это
   `TELEGRAM_BOT_TOKEN`.
2. Напиши своему новому боту любое сообщение (иначе он не сможет тебе отвечать).
3. Узнай свой **chat_id**: напиши боту **@userinfobot** — он пришлёт число. Либо
   открой `https://api.telegram.org/bot<ТОКЕН>/getUpdates` и найди
   `"chat":{"id":...}`. Это `TELEGRAM_CHAT_ID`.

## Развернуть в облаке с нуля

Если нужно поднять всё заново (например, на другом аккаунте GitHub):

1. Установи GitHub CLI и войди:
   ```powershell
   winget install --id GitHub.cli
   gh auth login            # разреши доступ к 'workflow'
   ```
2. Из папки проекта запусти скрипт (создаёт репозиторий, грузит код, переносит
   ключи из `.env` в секреты):
   ```powershell
   powershell -ExecutionPolicy Bypass -File .\deploy_to_github.ps1
   # приватный репозиторий:  ... deploy_to_github.ps1 -Visibility private
   ```
   Ключи читаются из `.env` на твоей машине и уходят прямо в зашифрованные секреты
   GitHub — в коде их нет (`.env` в `.gitignore`).

### Про приватность репозитория

- **Public** — минуты GitHub Actions бесплатны без лимита (рекомендуется; секретов
  в коде нет). Текущий репозиторий — публичный.
- **Private** — код скрыт, но бесплатно только 2000 минут/мес; на интервал 30 мин
  не хватит — увеличь интервал в `cron` до 1–2 часов.

### Другие варианты хостинга (без ПК)

| Вариант | Цена | Комментарий |
|---|---|---|
| **GitHub Actions** (текущий) | бесплатно (public) | Уже настроено. |
| **Oracle Cloud Free Tier** | бесплатно | Небольшой Linux-сервер 24/7. |
| **VPS** (Timeweb, Aeza и т.п.) | ~150–400 ₽/мес | Простой арендованный сервер. |
| **Raspberry Pi / мини-ПК** | разовая покупка | Стоит дома, мало энергии. |
